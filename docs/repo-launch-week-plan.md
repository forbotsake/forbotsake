# Repo Launch Week Plan (No X)

## Goal

Maximize GitHub stars in week 1 without relying on X as a distribution channel.

## Constraints

- X is read-only for now
- launch must work through repo packaging, HN, GitHub, direct founder sharing, communities, and secondary coverage

## Launch thesis

forbotsake should launch as the open-source answer to this problem:

"I built the product. Why is nobody paying attention?"

Not as:
- marketing skills for Claude Code
- prompt library
- AI copy helper

## Day 0: prep

Before launch:
- README must be launch-grade
- hero visual must exist
- examples must exist
- quickstart must work
- issue templates should be clean
- one HN post draft should be ready
- one founder-share message should be ready
- one follow-on proof post for day 2 should be ready

## Day 1: launch surfaces

Priority order:
1. GitHub repo itself
2. Hacker News / Show HN
3. Direct high-context sharing to relevant founders/builders/operators
4. Relevant communities where technical founders hang out
5. LinkedIn only if there is a sharp founder/operator angle

## Day 2-3

Publish or share:
- one concrete before/after example
- one artifact walkthrough
- one competitor contrast explanation

## Day 4-7

Keep the repo visibly alive:
- respond fast to issues
- polish README from real objections
- add one new example or proof asset
- post one more deeper breakdown in the best-performing surface

## What matters most

- one-line repeatable promise
- visible proof in seconds
- clear quickstart
- concentrated early burst
- follow-on proof loop
