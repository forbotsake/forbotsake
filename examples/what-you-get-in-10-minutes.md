# What You Get in 10 Minutes

This is the fastest way to understand the repo.

## Input

You start with:
- a real repo
- a rough product story
- a founder who can build but has weak distribution

## Run

```bash
/forbotsake-marketing-start
```

## Output

### 1. `strategy.md`
- positioning statement
- ICP
- channel strategy
- messaging pillars

### 2. `competitor-analysis.md`
- what competitors say
- where the whitespace is
- what angle is under-owned

### 3. `icp-profile.md`
- how the buyer talks
- where they hang out
- what they read
- what they actually care about

### 4. `content-calendar.md`
- what to publish
- where to publish it
- what each piece is trying to do

### 5. Draft assets
- launch copy
- content drafts
- outreach drafts
- review notes

## The point

The repo should not feel like “a few prompts that write posts.”
It should feel like a founder GTM system that produces real artifacts you can inspect and use.
